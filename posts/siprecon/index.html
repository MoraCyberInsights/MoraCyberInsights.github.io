Which vulnerability does the attack attempt to exploit? Link to heading

This attack did not exploit any specific CVE. This attack mostly highlights the reconnaissance phase of an attack.

Tactic: Discovery

Network Service Scanning (T1046)

The attacker is attempting to discover information about network services by sending SIP OPTIONS requests. This technique is used to identify open ports, services, and their configurations.

Link to heading

What is the goal of the attack? Link to heading

The activity is trying to gather information by doing reconnaissance trying to map out SIP servers and their configurations to identify potential targets for further exploitation.

If the system is vulnerable, do you think the attack will be successful? Link to heading

Yes. Given that this attack is part of the reconnaissance phase if the attacker finds a vulnerability, then it could be easily exploited for access.

How can a system be protected from this attack? Link to heading

Network Intrusion Prevention (M1031)

Use IDS/IPS to detect and block suspicious network traffic, including traffic patterns that may indicate scanning or probing activities.

Network Segmentation (M1030)

Implement network segmentation to limit access to sensitive systems. Ensure that SIP servers are placed in a segmented network with strict access controls to minimize exposure to unauthorized scanning.

Limit Access to Resource Over Network (M1035)

Set up firewall rules to restrict inbound and outbound traffic to limit access to SIP services. Only allow trusted IP addresses to communicate with SIP servers.

Filter Network Traffic (M1037)

Implement network filtering to block known malicious IP addresses and restrict protocols that are not required.
