Which vulnerability does the attack attempt to exploit? Link to heading

The attack does not exploit a CVE. However, it does take advantage of default account credentials and living off the land in order to attempt to complete it’s goal of extracting information from the endpoint.

What is the goal of the attack? Link to heading

This goal of this attack is to gather and exfiltrate information.

After establishing initial access using default credentials, the attacker proceeds to execute a series of commands by a script given the quick nature of the executed commands and their timestamps. All in a span of 2-3 seconds. The script enabled several commands that are common in a Linux environment. Then it executes the command to open a shell "sh" and executes "cat /proc/mounts; /bin/busybox LUJIH" to see the status of all mounted file systems as well as execute the custom applet LUJIH from busybox which is a software suite that provides several Unix utilities in a single executable file.

LUJIH: applet not found

The “LUJIH” applet that was called couldn’t execute because it was not in the local system. Then the script attempted to collect and exfiltrate information by executing the following:

cd /dev/shm; cat .s || cp /bin/echo .s; /bin/busybox LUJIH

dd bs=52 count=1 if=.s || cat .s || while read i; do echo $i; done < .s

/bin/busybox LUJIH

rm .s; exit

If the system is vulnerable, do you think the attack will be successful? Link to heading

Yes. With the attacker knowing what default credentials to use the attack can be successful. In this case it didn’t work because the custom applet for busybox was not in the local endpoint.
