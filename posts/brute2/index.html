Which vulnerability does the attack attempt to exploit? Link to heading

The attack attempts to exploit weak Account Use policies and Password policies. This was a Brute force attack. It was scripted based on the timeframe and speed of the attack.

From MITRE the tactics and techniques used were the following:

Reconnaissance

Active Scanning ID: T1595

Credential Access

Brute Force ID: T1110

Initial Access

Valid Accounts ID: T1078

What is the goal of the attack? Link to heading

The goal of the attack is to gain unauthorized access to the endpoint. It attempts to do so by brute forcing different usernames and passwords. This is probably obtained from a known passwords dictionary.

If the system is vulnerable, do you think the attack will be successful? Link to heading

Yes. If the system has weak Account Use policies set as well as weak password policies a brute force attack like this one could be successful.

How can a system be protected from this attack? Link to heading

Mitigations against Brute Force attacks:

ID M1036 Account Use Policies :

Set account lockout policies after a certain number of failed login attempts to prevent passwords from being guessed. Too strict a policy may create a denial of service condition and render environments un-usable, with all accounts used in the brute force being locked-out. Use conditional access policies to block logins from non-compliant devices or from outside defined organization IP ranges.

ID M1032 Multi Factor Authentication:

Use multi-factor authentication. Where possible, also enable multi-factor authentication on externally facing services.

ID M1027 Password Policies:

Refer to NIST guidelines when creating password policies.

ID M1018 User Account Management:

Proactively reset accounts that are known to be part of breached credentials either immediately, or after detecting bruteforce attempts.
